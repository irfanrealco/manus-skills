# Product Architecture: The 3-Layer Model

## Purpose

Design the exact product — not just the idea — before building anything. The 3-layer model forces clarity on what you're selling, how you deliver it, and what the buyer does next.

## Layer 1: Core Transformation

Define the before/after state the buyer experiences.

**Template:**
> "A [buyer persona] who [current painful state] becomes [desired outcome state] in [timeframe]."

**Example (Open Claw Concierge):**
> "A non-technical professional who has a broken or uninstalled Open Claw instance becomes someone with a fully operational, security-hardened AI agent in one 2-hour session."

The transformation statement is the product. Everything else — the features, the process, the price — is just evidence that the transformation is real.

**Common transformation patterns:**
| Pattern | Example |
|---|---|
| Broken → Working | Installation/setup services |
| Slow → Fast | Automation pipelines |
| Manual → Automated | Agent deployment |
| Confused → Clear | Decision frameworks |
| Generic → Specialized | Custom AI configuration |
| Isolated → Integrated | Stack integration services |

## Layer 2: Delivery Mechanism

Choose how the transformation is delivered:

| Mechanism | Best For | Scalability | Price Range |
|---|---|---|---|
| **Live service session** | High-trust, complex setup | Low (your time) | $299–$2,999 |
| **Done-for-you async** | Repeatable configurations | Medium | $199–$1,499 |
| **Digital download** | Templates, frameworks, scripts | High | $29–$299 |
| **Deployed agent** | Ongoing automation | Medium | $499–$4,999 |
| **Course/workshop** | Teaching the skill | High | $99–$999 |
| **SaaS/tool** | Recurring value delivery | Very high | $49–$499/mo (avoid unless intentional) |

**The Two-Tier Rule:**
Every product line must have exactly two tiers:
- **Entry tier** ($99–$599, one-time): Gets the buyer a quick win. Low commitment, high value density.
- **Expansion tier** ($999–$4,999, one-time): Builds on the entry win. Higher commitment, transformational outcome.

Never launch with a single offer. The entry tier creates buyers; the expansion tier creates revenue.

## Layer 3: Expansion Path

Define what the buyer buys next. This is not upselling — it is logical progression.

**Expansion path patterns:**
- **Depth**: Entry = basic setup → Expansion = advanced configuration + custom skills
- **Breadth**: Entry = one agent → Expansion = full agent team
- **Autonomy**: Entry = done-with-you → Expansion = done-for-you
- **Vertical**: Entry = general tool → Expansion = industry-specific version

**The Flywheel Test:**
Ask: "Does buying the entry product make the expansion product more desirable?" If yes, the expansion path is correct. If no, redesign.

## Product Architecture Document Template

```
PRODUCT: [Name]
VERTICAL: [Industry/niche]

CORE TRANSFORMATION:
Before: [painful state]
After: [desired state]
Timeframe: [how long it takes]

ENTRY TIER:
Name: [product name]
Price: $[amount] (one-time)
Delivery: [mechanism]
What they get: [3-5 specific deliverables]
Time to deliver: [hours/days]

EXPANSION TIER:
Name: [product name]
Price: $[amount] (one-time)
Delivery: [mechanism]
What they get: [3-5 specific deliverables]
Prerequisite: [entry tier or standalone?]

EXPANSION PATH:
Entry → Expansion trigger: [what makes them want more?]
Flywheel test: [pass/fail + reasoning]
```
