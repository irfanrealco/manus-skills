# GitHub Template Evaluation Criteria

When evaluating GitHub templates for a new game project, use these criteria to select the best one:

## 1. Core Functionality

- **User Authentication**: Registration, login, sessions
- **Lobby System**: Game list, create/join games, lobby chat
- **Game Rooms**: Room management, player joining/leaving
- **Real-Time Multiplayer**: Socket.IO or WebSocket infrastructure
- **Database Integration**: PostgreSQL, MySQL, or MongoDB

## 2. Technical Quality

- **Code Quality**: Clean, organized, well-commented
- **Tech Stack**: Modern and standard (Node.js, React, etc.)
- **Dependencies**: Not too many, not too old
- **Architecture**: Clear separation of concerns (frontend, backend, game logic)
- **Test Suite**: Presence of unit or integration tests

## 3. Project Health

- **Stars**: Higher is generally better
- **Forks**: Indicates community interest
- **Last Commit**: Recent activity is a good sign
- **Issues**: Check for open issues and how they are handled
- **Contributors**: More than one contributor is a plus

## 4. Ease of Use

- **Live Demo**: A live demo is a huge plus
- **Documentation**: Clear README with setup instructions
- **Setup Complexity**: How easy is it to get running locally?
