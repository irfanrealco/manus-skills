# Brainstorming Questions for UX Overhaul

Use these question templates to discover user intent, design direction, and brand identity. **Ask one question at a time** to avoid overwhelming the user.

## Phase 1: Identify Pain Points

**Primary Goal:**
- What's the primary pain point you want to solve?
  - A) Users aren't engaging enough - Need to make it more engaging/addictive
  - B) Managers can't get insights quickly - Need better data visualization
  - C) Interface feels cluttered/confusing - Need to simplify and streamline
  - D) Platform looks too generic - Need stronger branding and visual identity
  - E) Something else - Tell me what's bothering you most

## Phase 2: Narrow Down Specific Issues

**For Cluttered/Confusing (Option C):**
- When you say "cluttered/confusing," which part bothers you most?
  - A) The navigation bar - Too many links, hard to find what I need
  - B) The [main page] - Too many options before taking action
  - C) The [list page] - Overwhelming, hard to scan
  - D) The overall layout - Pages don't feel cohesive
  - E) Multiple areas - Combination of several pages

## Phase 3: Brand Identity & Emotional Tone

**Emotional Direction:**
- For the [brand name] branding, what feeling should the interface evoke?
  - A) Professional & Corporate - Clean, minimal, trustworthy (enterprise SaaS)
  - B) Energetic & Competitive - Bold, dynamic, gamified (fitness app/leaderboard)
  - C) Focused & Efficient - Utilitarian, no-nonsense, get-in-get-out (training tool)
  - D) Rustic & Grounded - Earthy, natural tones, calm confidence
  - E) Tech-Forward & Modern - Sleek, gradient-heavy, cutting-edge AI vibe

**Follow-up for combinations:**
- "Option [X] with some strong [Y]" → Clarify proportions (70/30? 50/50?)

## Phase 4: Navigation Structure

**Navigation Pattern:**
- Which navigation approach feels right?
  - A) Persistent sidebar - Always visible left nav with icons + labels
  - B) Collapsible sidebar - Hamburger menu that slides out when needed
  - C) Top nav with mega-dropdown - Clean header with grouped sections
  - D) Bottom tab bar - Mobile-style tabs at bottom for core actions
  - E) Hybrid approach - Different layouts based on user role

## Phase 5: Simplification Strategy

**For cluttered pages:**
- How should we simplify the [page name]?
  - A) Smart defaults - Remember last settings, one-tap "Quick Start" button
  - B) Wizard flow - Step-by-step (1. Pick X → 2. Pick Y → 3. Pick Z)
  - C) Preset modes - "Beginner Mode," "Challenge Mode," "Random" buttons
  - D) Minimal selection - Just core options + big "Start" button
  - E) Progressive disclosure - Show basics, expand for advanced options

**For list/history pages:**
- How should we compress the [list name]?
  - A) Card grid - Compact cards showing key info (3-4 per row)
  - B) Grouped by time - Collapsible sections: "This Week (5)", "Last Week (12)"
  - C) Summary stats only - Show totals/averages, click to expand full list
  - D) Infinite scroll with filters - Show 10 at a time, filter buttons at top
  - E) Timeline view - Vertical timeline with dots, hover to see details

## Phase 6: Color Scheme & Visual Identity

**Visual Style:**
- Which color scheme and visual identity?
  - A) Dark mode with neon accents - Black background, electric highlights (cyberpunk)
  - B) Light mode with bold gradients - White/gray base, vibrant gradients (modern SaaS)
  - C) High contrast athletic - Deep navy + bright orange/yellow (Nike/Peloton)
  - D) Monochrome + one accent - Black/white/gray + single brand color (minimal)
  - E) Dual theme - Let users toggle between dark competitive and light professional

**Brand Asset Request:**
- "Do you have brand assets (logo, color palette, style guide) I should use?"
- If yes: "Please share them so I can match your exact branding"

## Phase 7: Icon & Label Strategy

**For bottom nav/tabs:**
- Which icon treatment fits best?
  - A) Outlined icons - Thin line icons that fill when active (minimal)
  - B) Solid icons - Filled icons with glow effect when active
  - C) Custom branded icons - Unique icon set incorporating brand geometry
  - D) Icon + label always visible - Both shown at all times (clearest)
  - E) Icon only, label on active - Space-efficient, label appears for selected tab

## Phase 8: Tab Count & Structure

**For navigation:**
- How many tabs should be in the [navigation type]?
  - A) 4 core tabs - [List core features], everything else in overflow menu
  - B) 5 tabs - [List features], + More (overflow menu)
  - C) 5 tabs with role-based 5th - [List features], + (Manager Dashboard OR Profile)
  - D) 6 tabs - [List all features]
  - E) Different for roles - Reps get 4 tabs, Managers get 5 tabs

## Phase 9: Layout Structure

**Overall Layout:**
- Which layout structure feels most [brand adjective]?
  - A) Full-screen pages - Each tab takes entire screen, no persistent headers
  - B) Fixed top brand bar - Small logo + user avatar at top, content below, tabs at bottom
  - C) Floating action button - Main action FAB always visible, tabs at bottom
  - D) Card-based layout - Everything in cards with subtle shadows, background visible

## Tips for Effective Brainstorming

1. **One question at a time** - Don't overwhelm with multiple questions
2. **Offer concrete options** - Give 4-5 specific choices, not open-ended
3. **Use analogies** - "Think Nike Training Club meets cutting-edge AI"
4. **Clarify combinations** - "Option B with some E" → Ask for proportions
5. **Request assets early** - Logo, colors, fonts needed before implementation
6. **Confirm understanding** - Summarize their vision before planning
