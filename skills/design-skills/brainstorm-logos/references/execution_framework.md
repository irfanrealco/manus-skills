# Execution Planning Framework

This framework guides the creation of actionable execution plans from strategic vision and pattern analysis.

## Principles of Effective Execution Plans

### 1. Match the Plan to the Operating System
The plan must align with how the person actually operates, not how they "should" operate:
- If they work best under pressure, build pressure into the plan
- If they need accountability, structure check-ins
- If they think in systems, show how pieces connect
- If they value sovereignty, give them ownership and choice

### 2. Sequence Matters
Not everything can be done simultaneously. Identify:
- **Prerequisites** - What must be built before other things can work?
- **Dependencies** - What relies on what?
- **Natural order** - What sequence matches their energy and capacity?
- **Proof points** - What needs to be proven before scaling?

### 3. Insane Organization
Plans should be structured with extreme clarity:
- Clear phases with defined objectives
- Week-by-week breakdowns for near-term work
- Specific deliverables and success metrics
- Revenue targets (if applicable)
- Accountability structures built in

### 4. Balance Vision and Pragmatism
The plan must honor both:
- **The long-term vision** - Where they're ultimately going
- **The immediate reality** - What they can actually do now
- **The bridge** - How short-term actions fund/build toward long-term vision

## Execution Plan Structure

### Phase-Based Architecture

Organize work into distinct phases:

**Phase Definition:**
- **Name** - Clear, memorable phase name
- **Duration** - Realistic timeframe
- **Objective** - What this phase accomplishes
- **Success Metrics** - How you know it worked
- **Deliverables** - Concrete outputs
- **Revenue/Resources** - Financial or resource implications

**Phase Sequencing:**
- Each phase builds on the previous
- No skipping phases (or clear explanation why)
- Each phase has clear exit criteria

### Week-by-Week Breakdown (for near-term phases)

For the current phase, break down into weeks:

**Week X: [Objective]**
- **Actions** - Specific tasks to complete
- **Deliverables** - What gets produced
- **Revenue** - Money in/out (if applicable)
- **Pressure creation** - Built-in deadlines or commitments
- **Notes** - Important context or warnings

### Accountability Mechanisms

Build in structures for staying on track:

**Weekly Check-Ins:**
- What did I build last week?
- What did I deliver?
- What revenue did I generate?
- What's blocking me?
- What am I building this week?

**Monthly Reviews:**
- Did I hit the month's objectives?
- Am I on track for phase success metrics?
- What needs to adjust?

**Non-Negotiables:**
- List the things that MUST happen
- These are the guardrails

## Challenge and Accountability

The plan should include direct challenges:

**Challenge Format:**
- State the challenge clearly
- Explain why it matters
- Ask for commitment or pushback
- Create space for them to challenge back

**Example:**
"Can you commit to [specific action] for the next [timeframe]? If yes, I'll help you build the detailed plan and hold you accountable. If no, tell me why. Because maybe I'm missing something."

## Pressure-Creation Strategies

For people who work best under pressure:

**Healthy Pressure:**
- Pre-sell to trusted people
- Set tight but achievable deadlines
- Make public commitments
- Create visible progress tracking

**Unhealthy Pressure (Avoid):**
- Over-promising to strangers
- Unrealistic timelines
- No buffer for problems
- Reputation-damaging commitments

## Decision Points

Build in explicit decision points:

**Format:**
- **Decision** - What needs to be decided
- **Timing** - When to decide (not before, not after)
- **Criteria** - How to evaluate the decision
- **If YES** - What happens next
- **If NO** - Alternative path

## Relationship Debt and Tribe Building

When planning includes relationship elements:

**Relationship Debt:**
- Who do they owe attention/value to?
- Can this be converted to proof-of-concept work?
- How to clear debt while building proof?

**Tribe Identification:**
- Who's already in the tribe?
- Who's tribe-adjacent?
- What's the criteria for tribe membership?
- How does the plan serve/build the tribe?

## Revenue Integration

When revenue is part of the plan:

**Revenue Targets:**
- Realistic ranges (not single numbers)
- Tied to specific deliverables
- Explained reasoning (not arbitrary)

**Pricing Strategies:**
- Match to their operating system
- Build in urgency if appropriate
- Protect reputation (proof before scale)

**Revenue Tracking:**
- Week-by-week revenue generation
- Cumulative totals
- What revenue funds (next phase, living expenses, etc.)

## The Meta-Layer

Always show how the execution plan connects to:
- **The deeper vision** - How this serves the long-term goal
- **The pattern analysis** - How this matches their operating system
- **The tribe** - How this builds/serves their people
- **The rebellion** - What this is really about (if applicable)
