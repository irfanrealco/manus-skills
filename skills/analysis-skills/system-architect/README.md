# System Architect Skill

Design complex systems from vision to validated architecture using pattern recognition, wisdom extraction, and challenge-driven validation.

## Quick Start

```
/system-architect to design [system description]
```

Or simply mention: "I want to build [system], can you help me architect it?"

## What This Skill Does

Transforms raw vision into production-ready system architecture through a rigorous 5-phase process:

1. **Deep Understanding** - Ask clarifying questions, gather context
2. **Pattern Recognition** - Identify behavioral, cognitive, strategic patterns
3. **Wisdom Extraction** - Pull out concentrated insights
4. **Strategic Structuring** - Design complete architecture
5. **Challenge & Validate** - Stress-test with hard questions

## When to Use

- Designing complex systems (multi-agent, distributed, organizational)
- Building from vision (concept → concrete architecture)
- Validating before building (ensure design is sound)
- Challenging assumptions (act as sovereign partner)
- Creating sellable systems (explainable and defensible)

## What You Get

At the end of the process:

1. **Pattern Analysis** - 5-10 major patterns identified
2. **Gems of Wisdom** - 8-12 concentrated insights
3. **Architecture Document** - Complete technical specification (500-1000 lines)
4. **Challenges Document** - 8-12 hard questions
5. **Execution Plan** (optional) - Week-by-week breakdown

## Example

**Input**: "I want to build an LDS-inspired AI agent hierarchy"

**Output**:
- 782-line architecture document
- 10 gems of wisdom
- 10 hard questions including "You might be solving the wrong problem"
- User response: "Holy shit you are right!!"

## Key Principle

**Great architecture survives hard questions.** Build systems that can defend themselves before writing a single line of code.

- Works best for complex systems, not simple features
- Requires user engagement

- Document everything
